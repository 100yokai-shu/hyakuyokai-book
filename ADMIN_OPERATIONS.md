# 妖怪大福帳 管理者・ユーザー運用ガイド

## 目標

同じWebアプリURLを複数人に配布しつつ、各メンバーの入力データは混ざらないようにします。

管理者はアプリ本体を更新できます。ユーザーは自分のスマホで自分の売上を入力します。

## 役割

### ユーザー

- Netlifyなどで公開されたURLをスマホで開く
- ホーム画面に追加して使う
- 自分のスマホで自分の売上だけ入力する
- 必要なら `JSONバックアップ` を保存する
- PCとスマホを同期したい場合だけSupabase同期を設定する

### 管理者

- GitHubリポジトリを管理する
- Netlifyの公開設定を管理する
- `index.html` / `style.css` / `app.js` などを更新する
- 更新前に動作確認する
- 必要に応じてユーザーへ「再読み込みしてね」と案内する

## データが混ざらない理由

このアプリは、基本データを各スマホのブラウザ内 `localStorage` に保存します。

同じURLを開いても、スマホAのデータとスマホBのデータは別々です。URLを共有しただけではデータは混ざりません。

Supabase同期を使う場合だけ注意が必要です。同じ `自分専用同期ID` を使うと同じクラウドデータを読み書きします。メンバーごとに別の同期IDを使ってください。

## 管理者が更新するときの基本ルール

1. `STORAGE_KEY` は変更しない
2. 既存データを削除する処理を追加しない
3. 保存データの形を変えるときは `loadState()` で古い形式を読み取れるようにする
4. 大きな変更前にはユーザーへJSONバックアップを案内する
5. `index.html` の `?v=数字` と `sw.js` の `CACHE_NAME` を上げる

## 更新しても既存データが消えない仕組み

アプリ本体の更新と、ユーザーが入力した売上データは別物です。

- アプリ本体: Netlifyから配信されるHTML/CSS/JS
- 入力データ: 各スマホのブラウザ内のlocalStorage

そのため、通常の機能追加や画面変更では既存データは消えません。

ただし、次の操作はデータ消失につながります。

- ブラウザのサイトデータを削除する
- アプリ内の `全データ削除` を押す
- `STORAGE_KEY` を変更する
- 保存データの読み込み処理を壊す

## GitHub + Netlifyでの更新フロー

1. 管理者がGitHub上でファイルを編集する
2. `Commit changes` する
3. Netlifyが自動でデプロイする
4. 公開URLに最新版が反映される
5. ユーザーは同じURLを開くだけで最新版を使える

軽い文言修正ならGitHub上の編集だけで対応できます。

## ファイルごとの役割

- `index.html`: 画面の文字・タブ・入力欄
- `style.css`: 色・余白・見た目
- `app.js`: 計算・保存・同期・グラフ
- `sw.js`: ホーム画面追加/PWAのキャッシュ
- `manifest.json`: ホーム画面アプリ名・アイコン設定
- `supabase.sql`: Supabase同期用テーブル

## 更新前チェック

最低限、次を確認します。

```sh
node --check app.js
node --check sw.js
```

ブラウザでは次を確認します。

- 売上入力できる
- 月間成績が表示される
- 年間成績グラフが表示される
- 商品管理で登録/編集できる
- データタブでJSONバックアップが出せる

## ユーザーに案内する文面例

アプリを更新しました。
入力済みデータは基本的に消えません。
表示が古い場合は、一度アプリを閉じて開き直してください。
不安な場合は、データタブからJSONバックアップを保存してから使ってください。
